🕷️ Основы Web Scraping

  
Когда API нет — данные можно извлечь из HTML-страниц. Вот базовый пайплайн:


  

    

      
1

      

        
Отправить HTTP-запрос

        
requests.get(url) → получить HTML-код страницы

      

    

    

      
2

      

        
Парсинг HTML

        
BeautifulSoup(html, 'html.parser') → дерево DOM

      

    

    

      
3

      

        
Поиск элементов

        
soup.find_all('div', class_='post') → список элементов

      

    

    

      
4

      

        
Извлечение данных

        
el.text, el['href'] → текст и атрибуты

      

    

  


  

    

      

      scraper.py · BeautifulSoup
    

    
import requests
from bs4 import BeautifulSoup

# 1. Получаем HTML
url = "https://example.com/news"
resp = requests.get(url)

# 2. Парсим
soup = BeautifulSoup(resp.text, "html.parser")

# 3. Ищем все заголовки новостей
titles = soup.find_all("h2", class_="news-title")

# 4. Извлекаем текст
for t in titles:
    print(t.text.strip())

🏗️ Скрейпинг реальных сайтов без API

  

    Большинство популярных сервисов — риелторские сайты, доски объявлений о работе, маркетплейсы —
    не предоставляют публичный API. Но данные на них огромны и ценны для исследований, аналитики и ML-моделей.
  


  
  

    
⚠️ Почему нельзя просто скачать всё сразу?

    

      

        
🚦 Rate Limiting

        
Сервер считает запросы с одного IP. При превышении порога (~30–100 req/min) — блокировка 429 Too Many Requests или soft-ban на часы/дни.

      

      

        
🤖 Anti-bot системы

        
Cloudflare, DataDome, PerimeterX анализируют поведение: скорость, порядок запросов, TLS-fingerprint, mouse-movement. Боты распознаются даже с уникальными IP.

      

      

        
🍪 Сессии и куки

        
Многие сайты требуют авторизации или валидной сессии. Без cookies часть данных недоступна или приходит пустая страница.

      

      

        
⚡ JS-рендеринг

        
Контент загружается через React/Vue/Angular: requests.get() вернёт пустой HTML — данных нет. Нужен браузер, который выполнит JS.

      

      

        
🕵️ Honeypot-ловушки

        
Скрытые ссылки, невидимые обычному пользователю (CSS: display:none). Если бот переходит — IP моментально блокируется.

      

      

        
📋 CAPTCHA

        
reCAPTCHA v3 ставит оценку каждому запросу по поведению. Нет человеческого поведения — получаешь капчу или редирект на страницу верификации.

      

    

  


  
  
✅ Правильная стратегия «вежливого» парсера

  

    

      

        
1

        
Задержки между запросами — time.sleep(random.uniform(2, 5)). Случайный интервал имитирует человека лучше, чем фиксированный.

      

      

        
2

        
Ротация User-Agent — использовать список реальных строк браузеров (Chrome, Firefox). Один UA = явный признак бота.

      

      

        
3

        
Уважай robots.txt — urllib.robotparser или просто читай файл /robots.txt перед стартом. Часть страниц явно запрещена.

      

      

        
4

        
Обработка ошибок — при 429/503 не ломиться повторно. Exponential backoff: подождать 10 сек, 20 сек, 40 сек...

      

      

        
5

        
Сохранять прогресс — писать в файл каждые N записей. При вылете не начинать сначала.

      

      

        
6

        
Для JS-сайтов — Playwright/Selenium — запускает реальный браузер (Chromium), выполняет JS, ждёт нужных DOM-элементов.

      

    

  


  
  
🔧 Как работать с JS-рендерингом?

  
Современные сайты (React, Vue, Angular) не отдают данные в HTML — они загружают их через JS после открытия страницы. Есть два подхода:


  

    

      
🔍 Подход 1: Перехватить XHR/Fetch

      
Открой DevTools → Network → XHR/Fetch. Сайт сам дёргает свой внутренний API — ты видишь URL и ответ.

      

        
1. F12 → вкладка Network → фильтр Fetch/XHR

        
2. Обновить страницу → найти запрос с JSON-данными

        
3. Скопировать URL + заголовки (Authorization, Cookie)

        
4. Воспроизвести запрос через requests с теми же headers

      

    

    

      
🤖 Подход 2: Playwright (headless browser)

      
Запускает настоящий Chromium, выполняет весь JS, ждёт нужного селектора — потом читает DOM.

      

        
playwright_example.py

        
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
  browser = p.chromium.launch(headless=True)
  page = browser.new_page()
  page.goto("https://example.com")
  page.wait_for_selector(".listing-card")
  items = page.query_selector_all(".listing-card")
  for el in items:
    print(el.text_content())
  browser.close()

realty_scraper.py · шаблон

      
import requests, time, random, csv
from bs4 import BeautifulSoup

HEADERS = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/122.0 Safari/537.36",
    "Accept-Language": "ru-RU,ru;q=0.9",
    "Referer": "https://example-realty.ru/"
}

def scrape_page(page_num):
    url = f"https://example-realty.ru/moscow/rent/?p={page_num}"
    resp = requests.get(url, headers=HEADERS, timeout=15)
    if resp.status_code != 200:
        print(f"Ошибка {resp.status_code} на стр. {page_num}")
        return []
    soup = BeautifulSoup(resp.text, "html.parser")
    results = []
    for card in soup.select(".offer-card"):  # ← замени на реальный селектор!
        results.append({
            "price": card.select_one(".price").get_text(strip=True),
            "area":  card.select_one(".area").get_text(strip=True),
            "addr":  card.select_one(".address").get_text(strip=True),
            "link":  card.select_one("a")["href"]
        })
    return results

with open("apartments.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.DictWriter(f, fieldnames=["price","area","addr","link"])
    writer.writeheader()
    for page in range(1, 20):
        rows = scrape_page(page)
        writer.writerows(rows)
        print(f"Страница {page}: {len(rows)} объявлений")
        time.sleep(random.uniform(3, 7))  # ← вежливая задержка

✅ Трюк: JSON внутри HTML

        
    
          
  • Ищи тег <script type="application/ld+json">
    • 
          
  • Или <script id="__NEXT_DATA__"> — Next.js прячет там весь стейт
    • 
          
  • Это JSON — просто json.loads(), никакого BeautifulSoup
    • 
          
  • Самый надёжный способ для SSR-сайтов
    • 
        

      

    

    

      
jobs_scraper.py · Next.js __NEXT_DATA__ трюк

      
import requests, json, time, random
from bs4 import BeautifulSoup

HEADERS = {"User-Agent": "Mozilla/5.0 (X11; Linux x86_64) Chrome/122.0 Safari/537.36"}

def get_next_data(url):
    """Извлекаем JSON из __NEXT_DATA__ — работает на Next.js сайтах"""
    resp = requests.get(url, headers=HEADERS, timeout=15)
    soup = BeautifulSoup(resp.text, "html.parser")
    tag  = soup.find("script", id="__NEXT_DATA__")
    if not tag:
        return None
    return json.loads(tag.string)

def get_ld_json(url):
    """Достаём структурированные данные Schema.org из ld+json"""
    resp = requests.get(url, headers=HEADERS, timeout=15)
    soup = BeautifulSoup(resp.text, "html.parser")
    tag  = soup.find("script", type="application/ld+json")
    if not tag:
        return None
    return json.loads(tag.string)

all_jobs = []
for page in range(0, 10):
    url = f"https://example-jobs.ru/search?q=python&page={page}"
    data = get_next_data(url)
    if data:
        vacancies = data.get("props", {}).get("pageProps", {}).get("vacancies", [])
        all_jobs.extend(vacancies)
        print(f"Страница {page}: {len(vacancies)} вакансий")
    time.sleep(random.uniform(2, 5))

Задание 3: Мониторинг цен товарного агрегатора

        
Прототип: DNS, Яндекс.Маркет, Wildberries, OZON, Amazon, eBay

      

    

    

      Цель: выбрать категорию (ноутбуки, видеокарты, смартфоны) и собирать цены раз в день, строить график динамики.

      Что мешает: JavaScript-цены (React), Bot-score (Wildberries), сплит-тесты (разные цены для разных пользователей),
      геолокация (цены различаются по регионам).

    •

    •

    • 
        

      

      

        
🤖 Playwright для React-цен

    •

    •

    • 
        

      

    

    

      
price_monitor.py · Playwright + перехват сетевых запросов

      
import json, csv, time
from datetime import datetime
from playwright.sync_api import sync_playwright

prices_log = []

def intercept_response(response):
    """Перехватываем JSON-ответы API магазина"""
    if "api/catalog" in response.url and response.status == 200:
        try:
            data = response.json()
            for item in data.get("items", []):
                prices_log.append({
                    "date":  datetime.now().strftime("%Y-%m-%d"),
                    "name":  item.get("name"),
                    "price": item.get("salePriceU", 0) // 100
                })
        except: pass

with sync_playwright() as p:
    browser = p.chromium.launch(headless=True)
    page = browser.new_page()
    page.on("response", intercept_response)
    page.goto("https://example-shop.ru/catalog/laptops/")
    page.wait_for_load_state("networkidle")
    for _ in range(5):
        page.keyboard.press("End")
        time.sleep(2)
    browser.close()

with open("prices.csv", "a", newline="") as f:
    w = csv.DictWriter(f, fieldnames=["date","name","price"])
    w.writerows(prices_log)
print(f"Сохранено {len(prices_log)} записей")

    

    

      💡 Совет: Запускай скрипт раз в день через cron (Linux) или Планировщик задач (Windows).
      Через неделю у тебя будет датасет с динамикой цен — отличная основа для ML-модели предсказания распродаж.
